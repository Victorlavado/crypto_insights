# Learnings

Conocimiento sintetizado a partir del feedback acumulado. Esto NO es un diario — es el destilado.

Estructura propuesta:

- **`signal-performance.md`** — Qué signals predicen bien, mal, o son ruidosos. Por archetype.
- **`archetype-rules.md`** — Reglas y umbrales por archetype. Se actualiza cada vez que una regla demuestra (in)validez con datos.
- **`pattern-library.md`** — Patrones recurrentes detectados (ej. "post-listing en Coinbase: 2-3 semanas de volatilidad pre-trend").
- **`anti-patterns.md`** — Setups que parecen buenos pero pierden dinero sistemáticamente.

Regla: una observación en `feedback/` no entra aquí hasta haberse repetido al menos 3 veces o haber sido validada con un caso histórico.
